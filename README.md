# 아웃리치 회계 도우미

공식 회계 Excel 템플릿과 회계교육자료의 규칙을 바탕으로 만든 Windows/macOS용 로컬 데스크톱 앱입니다. 프로젝트 데이터와 첨부파일은 하나의 로컬 `.barun` 파일에 저장되며 서버 데이터베이스는 사용하지 않습니다.

다운로드 홈페이지: [https://tangeroooo.github.io/accounting-assistant/](https://tangeroooo.github.io/accounting-assistant/)

## 구현된 핵심 규칙

- 지출은 공식 8개 카테고리 순서로 묶고, 같은 카테고리 안에서는 날짜순으로 정렬합니다.
- 영수증 번호는 카테고리가 바뀔 때마다 1번부터 다시 부여합니다.
- 제공된 Excel의 6개 시트, 서식, 수식과 영수증철 원본 폼을 보존합니다. 행이 부족할 때는 `국내-금전출납부`에만 기존 서식을 복제해 행을 추가합니다.
- 결제자·계좌·정산 여부는 앱 내부 검산용으로만 사용하고 공식 Excel 및 영수증철에는 출력하지 않습니다.
- 영수증철은 고정 그리드 없이 그림 크기에 따라 위→아래로 먼저 이어 붙고, A4 높이를 넘으면 다음 세로 열로 자동 재배치됩니다. 지출 번호·내용은 편집 화면에서만 확인하고 PDF에는 넣지 않습니다.
- 온라인 영수증을 선택하면 워드/파워포인트처럼 8개 크기 조절 핸들이 표시됩니다. 일반 핸들은 그림 자체를 비율대로 확대·축소하고, `자르기` 모드에서는 검은 자르기 핸들로 프레임을 바꾸고 그림을 직접 드래그해 보이는 위치를 맞출 수 있습니다. 편집은 비파괴 방식이며 PDF에도 동일하게 반영됩니다.
- 영수증 이미지는 파일 선택과 클립보드 붙여넣기(`⌘V`/`Ctrl+V`)를 모두 지원합니다. PDF를 첨부하면 모든 페이지를 PNG로 바꿔 편집하고 `.barun`에도 변환된 이미지만 저장합니다.
- 오프라인 영수증은 지출 입력에서부터 부착 공간을 여러 개 만들고 각각의 너비·높이를 정할 수 있습니다. 영수증철 편집에서도 선택한 공간을 복제·삭제·크기 조절할 수 있어 긴 실물 영수증을 여러 조각으로 나눠 붙일 수 있습니다.
- 날짜·금액 노란색 표시와 금액 옆 영수증 번호 표시는 출력 후 사용자가 직접 합니다.
- 교통비의 주유비 산정 증빙은 개별 영수증과 1:1로 묶지 않고 프로젝트 공통 자료로 관리합니다. 온라인 파일과 인쇄 후 붙일 오프라인 부착칸을 각각 여러 개 추가할 수 있습니다.
- 수입·지출·자동 검토를 한 화면에 표시하며 지출 입력 중에도 수입 − 지출 − 환입액 차액을 실시간 계산합니다.
- 환입액은 `max(팀별사역지원금 − 6. 팀별사역비 합계, 0원)`으로 계산합니다. 팀별사역비가 지원금을 넘으면 별도 주의 없이 초과분을 팀회비 충당액으로 표시합니다.
- 첫 번째 팀별사역비 행의 비고에는 지원금과 회비 충당액을 원본 예시 형식으로 자동 작성합니다. 사용자가 비고를 수정하거나 비워 두면 수동 내용을 보존하며, `지원금·회비 비고 자동 작성` 버튼으로 다시 자동 모드로 바꿀 수 있습니다.
- 금전출납부의 내용은 내보낼 때 `[교통비]`, `[팀별사역비]`처럼 공식 항목 태그를 자동으로 붙입니다. 주유비 실물 원본, 여행자보험 증빙, 교역자·선교사 선물 분류, 국내 헌금 30만원 기준도 자동 검토합니다.
- 회비는 1인당 회비 × 인원수로 계산합니다.
- 새 프로젝트는 기본정보 → 수입 → `.barun` 파일 저장 순서로 안내하며, 파일을 만든 뒤 변경사항을 자동 저장합니다.
- 개인 선결제자는 설정에서 미리 만들지 않고 지출 입력 중 이름을 적으면 자동 등록됩니다.
- 진행 현황에서 회계 입력·검토 → 영수증철 편집 → 정산 확인 순서를 안내합니다.
- 회계보고서 Excel과 영수증철 PDF 버튼은 프로젝트 열기·저장 버튼 옆에 있으며, 영수증철에는 별도 인쇄 버튼이 없습니다.
- 앱 시작 시 GitHub Releases의 새 버전을 확인하며, 사용자가 동의하면 현재 프로젝트를 저장한 뒤 서명된 업데이트를 설치하고 앱을 다시 시작합니다. 상단 버튼으로 수동 확인도 가능합니다.

## 프로젝트 파일

`.barun`은 `manifest.json`과 영수증·증빙 이미지를 한 파일에 담는 ZIP 기반 문서 패키지입니다. 다른 컴퓨터에서는 `.barun` 파일 하나만 옮겨 열 수 있습니다. 저장할 때는 같은 폴더의 임시 파일을 완성한 뒤 기존 파일과 교체하므로 저장 도중 중단되어도 이전 파일을 바로 덮어쓰지 않습니다. 업데이트 직전에는 같은 폴더에 `프로젝트명-업데이트전-백업.barun`도 만듭니다. 자세한 형식은 [docs/barun-project-format.md](docs/barun-project-format.md)를 참고하세요. 기존 `회계프로젝트.json`은 열기만 지원하며 다음 저장부터 `.barun`으로 전환합니다.

## 로컬 개발

```bash
npm install
npm run tauri -- dev
npm test
npm run build
cargo check --manifest-path src-tauri/Cargo.toml
```

macOS 설치 이미지:

```bash
TAURI_SIGNING_PRIVATE_KEY=~/.tauri/accounting-assistant-updater.key TAURI_SIGNING_PRIVATE_KEY_PASSWORD='' npm run tauri -- build --bundles app,dmg
```

Windows 설치 파일은 Windows PC에서 같은 소스를 체크아웃한 뒤 다음 명령으로 빌드합니다.

```powershell
npm install
npm run tauri -- build
```

macOS·Windows 설치본을 GitHub Releases에 만들고 자동 업데이트를 배포하는 절차는 [docs/releasing.md](docs/releasing.md)를 참고하세요.

## 프로젝트 구조

```text
accounting-assistant/
├── src/                 React UI와 회계 로직
├── src-tauri/           Rust/Tauri 데스크톱 백엔드
├── resources/           앱이 복사해 사용하는 보존용 Excel 템플릿
├── reference/           사용자 제공 교육자료 원본(Git 제외)
└── artifacts/           테스트·렌더링·설치본 검증 결과(Git 제외)
```

`reference/`와 `artifacts/`는 로컬 검토용이며 GitHub에는 업로드하지 않습니다. 앱 실행과 빌드에 필요한 공식 Excel 템플릿 사본은 `resources/accounting-template.xlsx`에 포함됩니다.

## 템플릿 보존 원칙

앱은 `resources/accounting-template.xlsx`를 읽어 새 결과 파일을 생성합니다. 리소스 템플릿 자체에는 쓰지 않습니다. 원본에서 결과 파일로 복사하는 방식이므로 반복 내보내기를 해도 템플릿은 변경되지 않습니다.
